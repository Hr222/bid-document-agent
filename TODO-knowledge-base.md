# 知识库整理 TODO

这个文档用于第一阶段资料分类，目标不是一次分得很细，而是先为后续的结构化入库和 RAG 接入打基础。

## 一、一级分类

第一阶段先按“知识库业务大类”来分，不按现在磁盘目录编号来分。建议先统一成下面 8 类：

- [ ] `A_公司基础资料`
  - 公司主体身份信息、工商信息、章程、简介、组织架构、基础证照
- [ ] `B_人员与组织资料`
  - 人员花名册、资格证书、签名样本、社保在职证明、估价师查询、部门岗位信息
- [ ] `C_资质荣誉与合规资料`
  - 资质证书、ISO、荣誉、无违规记录、监管检查结果、信用查询结果
- [ ] `D_经营数据与财务资料`
  - 财务报告、审计数据、纳税信息、业务统计、经营指标、合作数据
- [ ] `E_项目业绩与案例资料`
  - 历史评估报告、项目清单、业主评价、典型案例、过往服务记录
- [ ] `F_制度规范与方法资料`
  - 管理制度、收费标准、专业文章、作业规范、质量控制制度、保密制度
- [ ] `G_投标与市场资料`
  - 开标报价、入围结果、市场排名、银行合作材料、投标支撑材料
- [ ] `H_暂不入库或敏感资料`
  - 培训照片、Logo、重复扫描件、账号密码、纯宣传素材、低价值附件

每个目录或子目录建议补充下面四项，后面建库会直接用到：

- [ ] `资料类型`
- [ ] `主要用途`
- [ ] `是否经常更新`
- [ ] `是否需要 OCR`

可直接按下面模板记录：

```text

资料类型：
主要用途：
是否经常更新：是 / 否
是否需要 OCR：是 / 否 / 部分需要
处理方式：结构化入表 / RAG全文检索 / 结构化+RAG / 暂不处理
备注：
```
