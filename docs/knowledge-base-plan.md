# 知识库整理计划

这个文档用于第一阶段资料分类与入库规划，目标不是一次分得很细，而是先为后续的结构化入库和 RAG 接入打基础。

## 一、一级分类

第一阶段先按“知识库业务大类”来分，不按当前磁盘目录编号来分。建议先统一成下面 8 类：

- `公司基础资料`
- `人员与组织资料`
- `资质荣誉与合规资料`
- `经营数据与财务资料`
- `项目业绩与案例资料`
- `制度规范与方法资料`
- `投标与市场资料`
- `暂不入库或敏感资料`

## 二、当前阶段重点

当前第一阶段重点不是全量资料，而是先围绕 RAG MVP 选择有代表性的样板资料。

当前优先样板：

- `18、公司制度`
- `收费标准`

原因：

- 能代表制度型知识库和规范型知识库
- 适合做全文检索、问答回溯和前端维护展示
- 能较快形成“入库 -> 检索 -> 问答 -> 验证”的闭环

## 三、第一阶段处理方式

第一阶段的处理方式建议采用下面几类：

- `结构化入表`
- `RAG 全文检索`
- `结构化 + RAG`
- `暂不处理`

当前建议：

- `18、公司制度` -> `RAG 全文检索`
- `收费标准` -> `结构化 + RAG`

## 四、首期排除内容

以下内容首期不纳入知识库：

- 纯图片资料
- 培训照片
- 重复盖章件
- 重复扫描副本
- 账号、密码、联系方式等敏感信息

## 五、当前阶段产出目标

第一阶段完成后，至少应该形成这些产出：

- 知识库表设计
- 文档到表结构的映射关系
- RAG 接入顺序
- 第一阶段 MVP 实施方案

## 六、和当前项目的关系

这份计划文档主要服务于当前第一阶段 MVP，也就是：

- 用制度样板和收费标准样板建立首批知识库
- 支持后续前端维护
- 支持 LLM 问答
- 支持后续召回率测试
