# 制度规范类知识库表设计说明

这份文档用于第一阶段的知识库表设计说明，当前重点面向 `制度规范与方法资料` 中具备“版本迭代”特征的资料类型。

## 适用范围

| 项目 | 说明 |
|---|---|
| 当前重点资料类型 | `管理制度`、`绩效考核` |
| 共同特征 | 多版本、可迭代、有启用/弃用状态、有审核流程、需要支持 RAG |
| 当前设计定位 | 第一阶段通用底座 |
| 后续扩展方式 | 在主档+版本底座上增加扩展表，不推翻主模型 |

## 设计目标

| 目标 | 说明 |
|---|---|
| 多版本管理 | 支持同一资料收录多个版本 |
| 版本迭代追踪 | 支持记录版本链、迭代次数、版本差异 |
| 生命周期管理 | 支持记录生效、失效、启用、弃用 |
| 当前版本定位 | 支持区分当前生效版本和最新收录版本 |
| 审核留痕 | 支持版本和章节级审核记录 |
| RAG 支撑 | 支持正文拆分、切块、向量检索 |

## 核心设计思路

| 层级 | 表名 | 作用 | 说明 |
|---|---|---|---|
| 主档层 | `kb_policy_document` | 记录资料本体 | 一条主档对应一项制度或绩效考核资料 |
| 版本层 | `kb_policy_version` | 记录具体版本 | 一条主档下面可以挂多个版本 |
| 块流层 | `kb_policy_block` | 记录文档 block 流 | 保存 OCR 来源、页码和原始顺序位置 |
| 正文层 | `kb_policy_section` | 记录结构化条文 | 按章、条、款拆分正文 |
| 检索层 | `kb_policy_chunk` | 记录 RAG 切块 | 用于向量检索和召回 |
| 审核层 | `kb_policy_review_record` | 记录审核动作 | 保存审核意见和时间 |
| 差异层 | `kb_policy_version_change` | 记录版本差异 | 支持版本对比与修订影响分析 |

## 主档表：`kb_policy_document`

主档表负责记录“这项资料本体是谁”，以及它现在对应哪个生效版本、哪个最新版本。

| 字段 | 含义 | 用途 |
|---|---|---|
| `policy_code` | 资料编码 | 用于唯一标识一项资料 |
| `policy_name` | 资料名称 | 例如《信息安全及保密制度》 |
| `policy_category` | 资料类别 | 例如 `管理制度`、`绩效考核` |
| `responsible_department` | 负责维护该资料的部门 | 便于责任归口和后续维护 |
| `current_version_id` | 当前生效版本ID | 回答“现在执行的是哪一版” |
| `latest_version_id` | 当前最新收录版本ID | 回答“知识库里最新收录的是哪一版” |
| `status` | 主档状态 | 表示资料本体当前是否处于有效管理状态 |

## 版本表：`kb_policy_version`

版本表负责记录“这项资料有哪些具体版本”，以及每个版本的来源、时间、生效状态和版本链关系。

| 字段 | 含义 | 用途 |
|---|---|---|
| `version_seq` | 版本顺序号 | 用于统计一项资料一共收录了多少代 |
| `version_label` | 版本显示名 | 例如 `2023版`、`2025版` |
| `source_year` | 来源年份 | 便于按年度筛选 |
| `source_document_date` | 原文件日期 | 记录资料原始版本时间 |
| `issued_at` | 发布/签发日期 | 表达发布时间 |
| `effective_date` | 生效日期 | 表达何时开始执行 |
| `expired_at` | 失效日期 | 表达何时停止使用 |
| `previous_version_id` | 上一版本ID | 用于构建版本迭代链 |
| `revision_type` | 修订类型 | 区分首版、常规修订、补充、废止等 |
| `version_status` | 版本状态 | 表达当前版本生命周期 |
| `change_summary` | 变更摘要 | 用于说明这一版改了什么 |
| `change_reason` | 修订原因 | 用于说明为什么改 |
| `source_path` | 源文件路径 | 记录原始文件位置 |
| `file_name` | 源文件名 | 记录文件名 |
| `file_hash` | 文件哈希 | 用于去重和校验 |
| `is_scanned` | 是否扫描件 | 区分原生文档和扫描件 |
| `parse_method` | 解析方式 | 例如 `direct`、`ocr` |
| `parser_status` | 解析状态 | 表达文本抽取进度 |
| `ingested_at` | 入库时间 | 记录进入知识库的时间 |
| `reviewed_at` | 审核完成时间 | 记录审核结束时间 |
| `approved_at` | 正式通过时间 | 记录正式通过时间 |

## 版本迭代设计

### 版本链设计

| 项目 | 说明 |
|---|---|
| 主字段 | `previous_version_id` |
| 作用 | 记录当前版本是从哪个上一版本演进而来 |
| 价值 | 支持版本链查询、版本差异比较、迭代次数统计 |
| 约束 | 必须指向同一 `policy_id` 下的版本 |

### 当前生效版本与最新收录版本

| 字段 | 含义 | 说明 |
|---|---|---|
| `current_version_id` | 当前生效版本 | 当前实际执行中的版本 |
| `latest_version_id` | 当前最新收录版本 | 知识库里最新收进来的版本，未必已启用 |

典型场景如下：

| 场景 | `current_version_id` | `latest_version_id` |
|---|---|---|
| 新版本尚未审核完成 | `2023版` | `2025版` |
| 新版本已启用 | `2025版` | `2025版` |

### 版本约束

| 约束 | 目的 |
|---|---|
| 同一资料下 `version_seq` 唯一 | 防止出现两个“第3版” |
| 同一资料下 `version_label` 唯一 | 防止出现两个同名版本 |
| `previous_version_id` 必须属于同一 `policy_id` | 防止跨资料串版本链 |
| `expired_at >= effective_date` | 保证生命周期时间合理 |
| `version_seq > 0` | 保证版本顺序号合法 |

## 状态与枚举说明

### 主档状态：`kb_policy_document.status`

| 枚举值 | 含义 | 使用场景 |
|---|---|---|
| `draft` | 主档已创建，但资料还没正式纳入可用知识库 | 刚建档、信息未补全 |
| `active` | 主档有效，正常参与检索、问答和版本管理 | 当前正常维护中的资料 |
| `archived` | 主档已归档，不再作为当前运营资料使用，但保留历史记录 | 整体停用、废止、合并后保留历史 |

补充说明：

| 项目 | 说明 |
|---|---|
| 管理对象 | 资料本体 |
| 不代表 | 某个具体版本一定生效 |

### 修订类型：`kb_policy_version.revision_type`

| 枚举值 | 含义 |
|---|---|
| `initial` | 首版 |
| `revise` | 常规修订 |
| `replace` | 整体替换 |
| `supplement` | 补充说明或补充条款 |
| `abolish` | 废止 |

### 版本状态：`kb_policy_version.version_status`

| 枚举值 | 含义 | 使用场景 |
|---|---|---|
| `draft` | 已入库，但还在整理 | 刚导入、未整理完成 |
| `reviewing` | 审核中 | 等待人工确认 |
| `approved` | 审核通过，但未正式启用 | 已通过，待切换 |
| `active` | 当前生效版本 | 正在执行/使用 |
| `superseded` | 已被新版本替代 | 历史版本，仍保留 |
| `retired` | 明确停用或废止 | 已退出使用 |

补充说明：

| 项目 | 说明 |
|---|---|
| 管理对象 | 某个具体版本 |
| `document.status = active` | 不等于所有版本都生效 |
| 真正执行中的版本 | `version_status = active` |

### 解析状态：`kb_policy_version.parser_status`

| 枚举值 | 含义 |
|---|---|
| `pending` | 待解析 |
| `processing` | 解析中 |
| `parsed` | 解析完成 |
| `failed` | 解析失败 |

### 章节审核状态：`kb_policy_section.review_status`

| 枚举值 | 含义 |
|---|---|
| `pending` | 待审核 |
| `reviewing` | 审核中 |
| `passed` | 审核通过 |
| `rejected` | 审核驳回 |

### 审核对象类型：`kb_policy_review_record.target_type`

| 枚举值 | 含义 |
|---|---|
| `document` | 主档 |
| `version` | 版本 |
| `section` | 章节 |
| `chunk` | 检索切块 |

### 审核动作：`kb_policy_review_record.review_action`

| 枚举值 | 含义 |
|---|---|
| `submit` | 提交审核 |
| `approve` | 审核通过 |
| `reject` | 审核驳回 |
| `return` | 退回修改 |
| `activate` | 启用 |
| `retire` | 停用/归档 |

### 版本差异类型：`kb_policy_version_change.change_type`

| 枚举值 | 含义 |
|---|---|
| `add` | 新增 |
| `update` | 修改 |
| `delete` | 删除 |
| `replace` | 替换 |
| `restructure` | 结构重组 |

### 版本差异范围：`kb_policy_version_change.change_scope`

| 枚举值 | 含义 |
|---|---|
| `document` | 整篇文档 |
| `chapter` | 章 |
| `section` | 节/条 |
| `clause` | 具体条款 |
| `metadata` | 元数据 |

### 版本差异影响级别：`kb_policy_version_change.impact_level`

| 枚举值 | 含义 |
|---|---|
| `low` | 低影响 |
| `medium` | 中影响 |
| `high` | 高影响 |

## 审核与检索相关表

| 表名 | 作用 | 说明 |
|---|---|---|
| `kb_policy_section` | 正文拆分 | 按章、条、款拆分，便于逐段审核、条款定位和差异对比 |
| `kb_policy_chunk` | RAG 切块 | 保存切块文本、所属版本、所属章节、检索元数据和向量 |
| `kb_policy_review_record` | 审核留痕 | 记录审核动作、审核意见、审核人和审核时间 |
| `kb_policy_version_change` | 版本差异 | 记录两个版本之间的差异摘要和影响级别 |

## 推荐查询场景

| 场景 | 说明 |
|---|---|
| 查询某项资料当前生效版本 | 用于回答“当前执行的是哪一版” |
| 查询某项资料最新收录版本 | 用于回答“最新收进来的是哪一版” |
| 查询某项资料一共迭代了多少次 | 用于统计版本代数 |
| 查询某项资料的完整版本链 | 用于查看历次版本演进 |
| 查询某个年份有哪些资料版本 | 用于按年度检索 |
| 查询 `2025版` 相较 `2023版` 改了什么 | 用于版本差异分析 |
| 查询某一条款原文 | 用于精确条款检索 |
| 查询待审核版本或章节 | 用于审核任务管理 |

## 当前定位

| 项目 | 说明 |
|---|---|
| 当前定位 | 第一阶段通用底座设计 |
| 当前重点服务对象 | `管理制度`、`绩效考核` |
| 后续扩展方式 | 如果需要收费项、收费区间、计费规则等细字段，在此底座上增加扩展表 |
